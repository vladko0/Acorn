# ACORN Admin

Админ менюто управлява публичния каталог в `data/products.json` и записва
новите изображения в `images/uploads/`.

## Качване На PHP Хостинг

Качете целия сайт, включително:

- `admin/`
- `api/`
- `data/products.json`
- `images/` и `images/uploads/.htaccess`
- публичните HTML, CSS и JavaScript файлове

При тестова инсталация в поддиректория адресът е например:
`https://acorn-bg.com/test/admin/`.

Хостингът трябва да изпълнява PHP. Не са нужни Node.js или rewrite правила,
защото админ панелът заявява директно файлове като `api/admin/login.php`.

Уверете се, че PHP има право да записва в:

- `data/`, за да променя `products.json`
- `images/uploads/`, за да добавя продуктови снимки

Файлът `api/config.php` съдържа само хеш на зададената админ парола. Файлът
`.env` е нужен само за локалния Node вариант и не трябва да се качва.

## Възможности

- Вход с администраторска парола и PHP сесия.
- Редакция и добавяне на продукти.
- Избор на опаковка (`пакет`, `чувал`, `кашон`) и тегло.
- Въвеждане на цена за килограм в EUR и автоматична стойност в лева по курс `1 EUR = 1.95583 BGN`.
- Информативно изчисление на общата цена на избраната разфасовка, без публикуването й в продуктовата карта.
- Филтър по категории.
- Прикачване на `JPG`, `PNG` или `WEBP` снимка до 5 MB; новата снимка се преобразува до квадратен `800 x 800` WEBP файл.

## Локално Тестване

За локалния preview може да продължите да използвате:

```powershell
node server.js
```

Node сървърът обслужва същите PHP-подобни API адреси за целите на локалната
проверка. PHP изпълнението се проверява след качване на хостинга.
